What is RSSV Workflow?

As the name already says RSSV Workflow is a front-end environment that facilitates the creation of your application with a series of preconfigured tools. We have as example a local server that recharges automatically every time you change a css, html or javascript file.

What have inside the RSSV Workflow?

Some Tools inside RSSV:

  • Html
  • Sass
  • Babel
  • Gulp
  • Browser Sync

Can you contribute?

Of course you can contribute. But have some rules with we need to follow for a better organization. The rules:

  • Open a issue, making a description of what feature you are going to improve or change
  • Push changes in a new branch , this branch must to have the name of the issue and need to be nested the paste issues.

    • Example: Issues/15

  • after commited and pushed, request a merge.
  • wait your request be answered, If accepted, your name will be included in the contributors section.
